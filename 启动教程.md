# 言象手语翻译系统 - 启动教程

> 本教程面向非技术人员，一步一步操作即可运行系统

---

## 一、项目概述

**言象** 是一个手语翻译系统，可以将手语视频翻译成文字。

**工作流程：**
```
手语视频 → 前端网页上传 → 后端服务器 → AI模型翻译 → 返回文字结果
```

---

## 二、前置准备

### 2.1 安装 Python

1. 打开浏览器，访问：https://www.python.org/downloads/
2. 点击 **Download Python 3.11.x**（或最新版本）
3. 下载完成后，双击安装文件
4. **重要**：安装时勾选 ✅ **Add Python to PATH**
5. 点击 **Install Now**

### 2.2 验证 Python 安装

按 `Win + R`，输入 `cmd`，回车，打开命令提示符：

```cmd
python --version
```

应该显示 `Python 3.11.x` 或类似版本号。

---

## 三、项目文件准备

### 3.1 获取项目文件

将项目文件夹（`prd_minimal`）放到合适的位置，例如：
```
C:\Users\1\Desktop\工作\slt\prd_minimal
```

### 3.2 验证虚拟环境

项目已自带 `venv` 虚拟环境（包含所有依赖），无需额外安装。

如需在新电脑首次运行，先验证 Python 是否安装，然后直接进入第四章启动即可。

---

## 四、启动后端服务

### 4.1 启动服务器

1. 打开命令提示符（`Win + R` → 输入 `cmd` → 回车）

2. 进入项目目录并启动服务器：
```cmd
cd C:\Users\1\Desktop\工作\slt\prd_minimal

set MOCK_MODE=false

.\venv\Scripts\python.exe server.py
```

3. 如果看到以下信息，说明启动成功：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
[言象] 启动完成 | MOCK_MODE=False
```

**保持这个窗口不要关闭！** 关闭后服务就停了。

### 4.2 验证服务状态

保持上面的服务器运行，另开一个命令提示符窗口（`Win + R` → `cmd` → 回车），输入：

```cmd
curl http://localhost:8000/health
```

如果返回 `{"status":"healthy"}`，说明服务正常运行。

## 五、访问前端网页

### 5.1 打开网页

直接双击打开 `index.html` 文件，或在浏览器中打开：

```
C:\Users\1\Desktop\工作\slt\prd_minimal\index.html
```

### 5.2 界面说明

打开后会看到一个分左右两边的界面：
- **左侧**：上传手语视频的区域
- **右侧**：显示翻译结果的区域

---

## 六、使用系统进行翻译

### 6.1 上传视频

1. 点击左侧的虚线框区域，或直接拖拽视频文件到该区域
2. 选择一个手语视频文件（支持格式：`.mp4`、`.mov`、`.avi`、`.mkv`）

### 6.2 等待处理

上传后会出现一个加载动画，右侧会显示处理进度（0% → 100%）。

**注意**：根据视频长短和网络状况，可能需要 **30秒到几分钟** 不等，请耐心等待。

### 6.3 查看结果

处理完成后，右侧会显示：
- **翻译文本**：手语对应的文字内容
- **置信度**：翻译的可信度百分比
- **响应时间**：处理所花费的毫秒数

## 七、常见问题

### 问题1：点击上传没反应

**原因**：后端服务没有启动
**解决**：
1. 确认服务器窗口是否打开
2. 确认服务器窗口中没有报错信息
3. 重新运行启动命令（第四章）

### 问题2：一直显示 95% 不动

**原因**：可能是网络问题或 API 响应超时
**解决**：
1. 检查服务器窗口是否有错误信息
2. 等待 2 分钟看是否恢复
3. 刷新页面后重试

### 问题3：显示"网络通信错误"

**原因**：
- 后端服务已停止
- 端口被占用
**解决**：
1. 关闭所有命令提示符窗口
2. 重新按照第四章启动服务器
3. 刷新网页重试

### 问题4：端口被占用

**错误信息**：`Port 8000 is already in use`

**解决**：
1. 关闭所有使用 8000 端口的程序
2. 或修改 server.py 第 198 行的端口为其他值（如 8001）

### 问题5：翻译结果为乱码或错误

**原因**：这是 AI 模型的问题，不影响程序运行
**解决**：这是正常现象，模型效果取决于视频质量和手语内容

## 八、如何关闭系统

1. 关闭运行服务器的 **命令提示符窗口**
2. 关闭浏览器标签页
